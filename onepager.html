The reframe
Not a document problem — a fragmentation problem. The context already exists; it's scattered across five systems and stitched by hand, the morning of. So the brief should assemble itself and land in Slack the day before — with a human in the loop.

🔴 Today

  • Staffer manually queries Salesforce, email, Slack, docs & web — one by one
  • Done the morning of, under time pressure
  • ~30–45 min per meeting; quality depends on who does it
  • Knowledge trapped in one person's head

🟢 With Partner Briefing

  • Triggers off the calendar event — no one runs anything
  • Brief waiting in Slack the day before
  • ~2 min to read; consistent structure every time
  • Human reviews & refines; corrections tune the agent

What makes it trustworthy

🔗Provenance — every claim cites its source (Salesforce / email / Slack / docs / web). Real vs. illustrative data is labelled in the UI.
⚠️Confidence flags — the agent surfaces what's thin or stale instead of bluffing. It admits what it doesn't know.
🧑‍⚖️Human-in-the-loop — the agent drafts; a person signs off. Corrections feed back and tune future briefs.
🔐Respects permissions — only surfaces what the viewer can already access. Never widens access.

How I'd sequence it — 90 days

Wk 1–3
Shadow + map
Sit with 3–4 staffers; map what they pull & where it breaks. Validate before building.
Wk 3–6
Thin slice
Salesforce-only briefs for one team. Prove the structure earns trust.
Wk 6–10
Add sources + Slack
Layer email, Slack, web. Ship AgentForce → Slack delivery + feedback loop.
Wk 10–13
Measure + expand
Track adoption, time saved, edit-rate. Roll out as briefs need fewer corrections.
